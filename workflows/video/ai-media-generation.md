# AI Media Generation Map

> Start here if you need to turn an idea into audio, video, dubbing, or publish-ready media and you want the shortest route to the right local workflow.

## What this is for

This repo had started to collect too many media tools in one place. That made the file noisy and hard to trust.

Use this page as a map:

- if you are making music, open a music workflow
- if you are making avatar or dubbed video, open a video workflow
- if you are cleaning or packaging audio, open the production workflow

## Start with the asset you need

| If you need to produce... | Open this |
|------|------|
| a song from an idea, hook, or lyric draft | [Suno Song to Release](../music/suno-song-to-release.md) |
| a cleaner production pipeline with stems, mastering, and packaging | [Music Production Automation](../music/music-production-automation.md) |
| an avatar-led video with translation or dubbing and clear consent rules | [AI Avatar and Dubbing with Consent](ai-avatar-and-dubbing-with-consent.md) |
| notes, clips, and repurposed assets from an existing recording | [Video Transcript to Content](video-transcript-to-content.md) |
| titles, descriptions, and publishing metadata for YouTube | [YouTube SEO Metadata](youtube-seo-metadata.md) |

## A good default stack

| Stage | Typical tool |
|------|------|
| idea and prompt drafting | LLM of choice |
| music generation | Suno |
| spoken-audio cleanup | Adobe Podcast or Descript |
| avatar video | HeyGen |
| final editing | your DAW or NLE |
| verification and disclosure | Content Credentials where available |

## Notes

- Prefer one strong workflow over a six-tool chain you cannot maintain.
- Keep the human review step where the content becomes public.
- If a workflow touches voice or likeness, use explicit consent and disclosure.
