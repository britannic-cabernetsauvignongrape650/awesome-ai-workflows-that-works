# Claude Code + GitHub Actions — Async Agent in CI

> Label a GitHub issue, wake up to a branch + implementation + PR. Claude Code runs in CI while you work on other things.

## What this is for

Instead of running Claude Code interactively in your terminal, this workflow triggers it automatically from GitHub Actions when an issue is labeled. Claude reads the issue, creates a branch, implements a solution, runs tests, and opens a PR — all asynchronously.

This replaces the now-deprecated OpenAI Codex Async workflow. Claude Code is the current standard for agentic coding in CI.

## How It Works

```
Issue labeled `agent:claude`
        ↓
GitHub Actions triggers
        ↓
Claude Code reads the issue body + codebase
        ↓
Creates branch: fix/issue-{number}-{slug}
        ↓
Implements solution (with TDD if CLAUDE.md specifies it)
        ↓
Runs test suite — must pass before PR
        ↓
Opens draft PR with implementation summary
        ↓
Human reviews and merges
```

## Setup

### 1. Store Anthropic API Key in GitHub Secrets

```
GitHub repo → Settings → Secrets and variables → Actions → New repository secret
Name: ANTHROPIC_API_KEY
Value: sk-ant-...
```

### 2. Create the Workflow File

`.github/workflows/claude-agent.yml`:

```yaml
name: Claude Code Agent

on:
  issues:
    types: [labeled]

jobs:
  claude-agent:
    # Only run when labeled with 'agent:claude'
    if: github.event.label.name == 'agent:claude'
    runs-on: ubuntu-latest
    permissions:
      contents: write
      issues: write
      pull-requests: write

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # full history for better context

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Configure git
        run: |
          git config user.name "claude-agent[bot]"
          git config user.email "claude-agent[bot]@users.noreply.github.com"

      - name: Run Claude Code on issue
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          ISSUE_NUMBER: ${{ github.event.issue.number }}
          ISSUE_TITLE: ${{ github.event.issue.title }}
          ISSUE_BODY: ${{ github.event.issue.body }}
        run: |
          # Create branch
          BRANCH="fix/issue-${ISSUE_NUMBER}-$(echo "${ISSUE_TITLE}" | tr '[:upper:]' '[:lower:]' | tr ' ' '-' | tr -cd '[:alnum:]-' | cut -c1-40)"
          git checkout -b "$BRANCH"

          # Run Claude Code
          claude -p "
          You are fixing GitHub issue #${ISSUE_NUMBER}.

          Issue title: ${ISSUE_TITLE}
          Issue description:
          ${ISSUE_BODY}

          Instructions:
          1. Read the relevant code files to understand the codebase
          2. Implement a complete fix for this issue
          3. Follow existing code style and conventions (check CLAUDE.md if present)
          4. Write or update tests to cover the fix
          5. Run the test suite and ensure all tests pass
          6. Commit your changes with message: 'fix: ${ISSUE_TITLE} (closes #${ISSUE_NUMBER})'

          Do not open a PR — the CI workflow will do that.
          " 2>&1

          # Push branch
          git push origin "$BRANCH"

          # Open draft PR
          gh pr create \
            --title "fix: ${ISSUE_TITLE}" \
            --body "Resolves #${ISSUE_NUMBER}

          ## What Changed
          This PR was generated by Claude Code based on issue #${ISSUE_NUMBER}.

          ## Review Notes
          - Check that the implementation matches the issue requirements
          - Verify test coverage is adequate
          - Review for edge cases not mentioned in the issue

          🤖 Generated by Claude Code" \
            --draft \
            --head "$BRANCH" \
            --base main

      - name: Comment on issue
        if: success()
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          ISSUE_NUMBER: ${{ github.event.issue.number }}
        run: |
          gh issue comment "$ISSUE_NUMBER" --body "🤖 Claude Code has created a PR for this issue. Please review it before merging."

      - name: Comment on failure
        if: failure()
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          ISSUE_NUMBER: ${{ github.event.issue.number }}
        run: |
          gh issue comment "$ISSUE_NUMBER" --body "❌ Claude Code failed to implement this issue automatically. Manual implementation required. Check the Actions run for details."
```

### 3. Add a CLAUDE.md to Your Project

Claude reads this at startup. Without it, Claude has no context about your conventions.

```markdown
# CLAUDE.md

## Project
[Brief description — 2 sentences]

## Stack
- Language/runtime versions
- Key frameworks and libraries
- Test framework

## Conventions
- Branch naming: fix/[issue-number]-[slug]
- Commit format: Conventional Commits
- Test location: src/__tests__/

## Rules
- Run tests before committing
- Never push directly to main
- All new functions need unit tests
```

### 4. Label Issues to Trigger the Agent

```bash
# From CLI
gh issue edit 42 --add-label "agent:claude"

# Or from GitHub UI: Issues → select issue → Labels → agent:claude
```

## Advanced Configuration

### Restrict to Specific Labels and Repos

Only trigger on issues with both `agent:claude` AND `good-first-issue`:

```yaml
if: |
  contains(github.event.issue.labels.*.name, 'agent:claude') &&
  contains(github.event.issue.labels.*.name, 'good-first-issue')
```

### Add Budget Control

Prevent runaway costs by limiting Claude's execution time:

```yaml
- name: Run Claude Code on issue
  timeout-minutes: 15  # kill after 15 minutes
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

### Require Passing Tests Before PR

```yaml
- name: Run test suite
  run: npm test  # or pytest, cargo test, etc.

- name: Open PR (only if tests pass)
  if: success()
  run: gh pr create ...
```

### Scope Claude to Specific Directories

For security, restrict what Claude can access by configuring allowlists in `.claude/settings.json`:

```json
{
  "permissions": {
    "allow": ["Read", "Write", "Edit"],
    "deny": []
  }
}
```

Also scope the task by being explicit in the prompt: "Only modify files in `src/` and `tests/`."

## Failure modes

- **Issue quality matters most.** Vague issues ("button broken") produce poor implementations. Good issues have: expected behavior, actual behavior, reproduction steps.
- **Context window on large repos.** Claude may miss relevant files in very large codebases. Add file paths in the issue body: "The relevant code is in `src/api/search.ts`."
- **Always review draft PRs.** Claude opens draft PRs deliberately — never auto-merge. Review the diff before marking ready.
- **Secrets security.** The `ANTHROPIC_API_KEY` is never exposed in logs if properly stored as a secret, but review your Actions permissions setup carefully.
- **Test flakiness.** If your test suite has flaky tests, Claude may loop trying to fix tests that aren't actually broken by its change.
