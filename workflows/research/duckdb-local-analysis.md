# DuckDB + Claude — Local Data Analysis

> Analyze CSV, Parquet, JSON, and Excel files locally with DuckDB. No server, no cloud, no data leaving your machine.

## What this is for

DuckDB is an in-process analytical database that reads files directly from disk — no server required. With Claude Code and the DuckDB skill, it becomes a conversational analysis tool: describe what you want in English, get correct DuckDB SQL on the first try, see results immediately.

Ideal for: ad-hoc analysis of exported CSVs, analyzing data files before loading them into a proper DB, quick EDA (exploratory data analysis), one-off business questions.

## Setup

### 1. Install DuckDB

```bash
# Python (most common)
pip install duckdb

# Or via CLI
brew install duckdb          # macOS
winget install DuckDB.cli    # Windows
```

### 2. Install Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

Claude Code works with DuckDB out of the box — just describe your data and ask questions in natural language. For better results, add a CLAUDE.md noting that DuckDB is installed and listing your data files.

## Usage

### Basic: Drop Files and Ask

```bash
# Place your data files in the working directory
ls
# sales_2025.csv  orders.parquet  customers.json

# Ask Claude in natural language
claude "
I have these files in the current directory:
- sales_2025.csv: columns date, product_id, amount, region, salesperson
- customers.json: columns customer_id, name, segment, acquisition_date

Analyze the sales data:
1. Monthly revenue trend for 2025
2. Top 10 salespeople by total revenue
3. Which region has the highest average deal size?
4. Flag any months where revenue dropped more than 15% MoM
"
```

### What Claude Does

```sql
-- Generated by Claude with DuckDB skill

-- 1. Monthly revenue trend
SELECT
    DATE_TRUNC('month', CAST(date AS DATE)) AS month,
    SUM(amount) AS revenue,
    LAG(SUM(amount)) OVER (ORDER BY DATE_TRUNC('month', CAST(date AS DATE))) AS prev_month,
    ROUND(
        (SUM(amount) - LAG(SUM(amount)) OVER (ORDER BY DATE_TRUNC('month', CAST(date AS DATE))))
        / LAG(SUM(amount)) OVER (ORDER BY DATE_TRUNC('month', CAST(date AS DATE))) * 100,
        1
    ) AS mom_pct_change
FROM 'sales_2025.csv'
GROUP BY 1
ORDER BY 1;

-- 2. Top 10 salespeople
SELECT salesperson, SUM(amount) AS total_revenue, COUNT(*) AS deals
FROM 'sales_2025.csv'
GROUP BY salesperson
ORDER BY total_revenue DESC
LIMIT 10;
```

Claude executes the queries, interprets the results, and flags the anomalies you asked for — without you writing a single line of SQL.

## DuckDB-Specific Features Worth Knowing

Claude with the DuckDB skill uses these correctly — they don't exist in standard SQL:

```sql
-- Read multiple files at once with glob
SELECT * FROM 'sales_*.csv';
SELECT * FROM read_parquet('data/**/*.parquet', hive_partitioning=true);

-- EXCLUDE specific columns
SELECT * EXCLUDE (internal_id, created_at) FROM 'orders.csv';

-- COLUMNS(*) for patterns
SELECT COLUMNS('.*_amount') FROM 'orders.csv';  -- all *_amount columns

-- QUALIFY window functions (no subquery needed)
SELECT *, ROW_NUMBER() OVER (PARTITION BY region ORDER BY amount DESC) AS rank
FROM 'sales.csv'
QUALIFY rank <= 5;

-- Struct and list operations on JSON
SELECT json_col->>'name', json_col->'tags'[0] FROM 'data.json';

-- Pivot tables (no CASE WHEN)
PIVOT orders ON region USING SUM(amount) GROUP BY product_id;
```

## Failure modes

- **Memory limits:** DuckDB keeps working data in memory. Very large files (10GB+) may require the persistent database mode (`duckdb my.db`) or explicit memory limits (`SET memory_limit='4GB'`).
- **Write operations:** DuckDB is OLAP (read-heavy analytics), not OLTP. Don't use it as an application database.
- **Excel files:** Install the Excel extension first: `INSTALL excel; LOAD excel;` then `SELECT * FROM read_xlsx('file.xlsx')`.
- **Encoding issues with CSV:** If you get parse errors, try `read_csv('file.csv', auto_detect=true)` or specify `encoding='latin1'` for older exports.
- **Date parsing:** CSV dates are read as VARCHAR by default. Always cast: `CAST(date_column AS DATE)`.
